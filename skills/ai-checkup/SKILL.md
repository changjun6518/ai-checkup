# AI 활용 건강검진 (AI Checkup)

사용자의 로컬 Claude Code 세션 로그를 분석해 "AI 활용 건강검진" 리포트를 만든다.
톤은 건강검진 소견서: 평가·채점이 아니라 셀프케어를 돕는 뉘앙스를 유지한다.
**모든 분석은 로컬에서만 이루어지며, 원본 프롬프트는 절대 외부로 전송하거나 공유 카드에 포함하지 않는다.**

## 진행 순서

### 1단계: 기계 지표 집계

아래 순서로 시도하고, 처음 성공한 것을 사용한다 (두 스크립트는 동일한 JSON을 출력함):

```bash
python3 "${CLAUDE_PLUGIN_ROOT}/skills/ai-checkup/analyze.py"   # 1순위
python "${CLAUDE_PLUGIN_ROOT}/skills/ai-checkup/analyze.py"    # 2순위 (Windows 등)
node "${CLAUDE_PLUGIN_ROOT}/skills/ai-checkup/analyze.js"      # 3순위 (npm 설치 환경)
```

(`CLAUDE_PLUGIN_ROOT`가 비어 있으면 이 SKILL.md와 같은 디렉토리의 스크립트를 실행)

셋 다 실패하면: jsonl을 직접 파싱하지 말고, 사용자에게 Python 또는 Node 설치가 필요하다고
안내하며 진단을 중단한다 (원본 로그 직접 읽기는 컨텍스트 낭비가 크고 수치가 부정확해짐).

출력 JSON의 `summary`(전체 집계)와 `sessions`(세션별 요약 + 샘플 프롬프트)를 읽는다.
세션이 3개 미만이면 "데이터가 부족해 참고용 진단"임을 리포트 상단에 명시한다.

### 2단계: 프롬프트 품질 채점 (LLM 채점)

각 세션의 `sample_first_prompt`와 `sample_corrections`를 읽고 아래 루브릭으로 채점한다.
**반드시 루브릭 기준으로만 채점하고, 각 점수마다 실제 프롬프트 인용을 근거로 남긴다.**

#### 채점 축 (각 1~5점)

**A. 첫 프롬프트 구체성** — 첫 요청만으로 작업이 시작 가능한가?
- 5: 목표 + 맥락 + 제약/조건이 모두 명시됨 (예: 구조화된 기획서 형태)
- 3: 목표는 분명하나 맥락이나 조건이 빠져 AI가 되물어야 함
- 1: "이거 고쳐줘" 수준, 대상조차 불명확

**B. 교정 능력** — AI가 빗나갔을 때 요구사항을 좁혀주는가?
- 5: 교정 시 무엇이 다른지 + 원하는 결과를 구체적으로 재정의
- 3: "아니 그게 아니라" 후 부분적 단서만 제공
- 1: 같은 말 반복 또는 감정적 반응
- 교정 상황이 없으면(correction_rate 0) 첫 프롬프트 품질로 대체 추정하고 "추정"임을 표기

**C. 모델-작업 매칭** — 작업 난이도에 맞는 모델을 쓰는가?
- `models`와 세션 내용을 대조: 최상위 모델(fable/opus)로 인사·단순질문·오타수정을 하면 감점
- `sample_trivial_on_expensive`가 비어있지 않으면 해당 인용을 근거로 제시

**D. 세션 위생** — 세션을 작업 단위로 관리하는가?
- compaction 0회 + 세션당 프롬프트 15개 이하면 가점
- 한 세션에 무관한 작업 여러 개가 섞여 있으면 감점 (세션 title과 프롬프트 대조)

**E. 고급 기능 활용** — plan mode, 스킬, 서브에이전트, CLAUDE.md를 쓰는가?
- `plan_mode_sessions`, `skills_used`, `top_tools`의 Agent/Skill 항목으로 판단

### 3단계: 유형 진단

점수 조합으로 유형을 하나 부여한다. 예시 (자유롭게 변형 가능):

- **한방 프롬프터** (A≥4, 교정 거의 없음): 첫 요청에 다 담는 타입
- **핑퐁 대화가** (A≤3, 프롬프트 수 많음): 짧게 여러 번 주고받는 타입
- **위임형 지휘관** (E≥4): 에이전트/스킬에 일을 뿌리는 타입
- **수동 장인** (E≤2, 툴 호출 대비 Edit/Bash 비중 높음): 직접 다 시키는 타입
- **화력 과잉** (C≤2): 모든 일에 최상위 모델을 쓰는 타입

### 4단계: 리포트 출력

아래 구성으로 출력한다:

리포트 타이틀은 "🩺 AI 활용 건강검진 결과"로 한다.

1. **유형명 + 종합 소견 한 줄** — 검진 소견서 톤. 예: "첫 프롬프트 밀도가 높아 교정 왕복이 발생하지 않는 건강한 사용 패턴입니다."
2. **5축 검진 결과표** (A~E, 각 점수 + 한 줄 근거)
3. **근거 인용** — 본인 프롬프트에서 잘한 예 1개, 아쉬운 예 1개 (있는 경우만)
4. **처방전** — 다음 주에 시도할 구체적 행동 2~3개
5. **공유 카드** — 마지막에 코드블록으로 제공. 카드 타이틀은 "🩺 AI 활용 건강검진".
   유형명 + 5축 점수 + 한줄평만 포함하고
   **원본 프롬프트, 프로젝트명, 세션 제목은 절대 넣지 않는다** (슬랙 공유용).

## 주의

- 점수는 루브릭 앵커에 맞춰 보수적으로. 데이터가 애매하면 중간 점수 + "표본 부족" 표기.
- 원본 jsonl을 직접 읽지 말 것 — 컨텍스트 낭비. 반드시 analyze.py 출력만 사용.
- 진단은 놀림이 아니라 코칭. 아쉬운 점도 개선 행동과 함께 제시.
