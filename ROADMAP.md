# AI 활용 건강검진 — 확장 설계 노트

v0.2.0 이후 검토할 평가 항목 후보와 성능/제한 설계. 코드 반영 전 아이디어 문서.

## 1. 확장 평가 항목 후보

### 기계 측정 가능 (스크립트만으로 계산, 신뢰도 높음)

| 항목 | 측정 방법 | 의미 |
|---|---|---|
| **맥락 첨부율** | 프롬프트 내 파일 경로·코드블록·에러 메시지 포함 비율 (정규식) | "이거 왜 안돼?" vs "이 에러 로그 첨부, X 파일에서 발생" |
| **산출물 형태 명시율** | "표로", "목록으로", "파일로 저장" 등 출력 형식 지시어 포함 비율 | 재작업을 줄이는 습관. 이번 진단 처방전 3번과 직결 |
| **첫 시도 폐기율** | 서로 다른 세션에 동일/유사 첫 프롬프트 반복 등장 | 세션을 버리고 다시 시작한 흔적 (실데이터에서 이미 1건 발견) |
| **중단 개입 빈도** | `[Request interrupted]` 레코드 수 | AI가 산으로 갈 때 빨리 끊는 능력. 너무 많으면 첫 지시 부족, 0이면 방치 가능성 — 양면 지표 |
| **캐시 효율** | cache_read / 전체 input 토큰 비율 | 세션을 연속으로 굴리는지, 뜸 들이다 캐시 다 날리는지 |
| **병렬 활용도** | 백그라운드 작업·서브에이전트 호출 수 | 기다리는 동안 다른 일 시키는 능력 |
| **환경 구축도** | CLAUDE.md 존재, 훅/MCP 설정, 커스텀 스킬 보유 여부 | 1회성 사용자 vs 시스템을 만드는 사용자 |
| **피드백 간격** | 프롬프트 간 timestamp 간격 분포 | 능동적 세션 관리 vs 방치 |

### LLM 채점 필요 (재미와 인사이트 담당)

| 항목 | 채점 방법 | 의미 |
|---|---|---|
| **세션 완주율** | 세션 마지막 왕복이 목표 달성 종료인지 판정 | 시작한 일을 끝내는가 |
| **위임 적절성** | 직접 지시 vs 에이전트 위임 선택이 작업 규모에 맞았는지 | 마이크로매니징 감지 |
| **질문 품질 성장** | 기간을 반으로 갈라 전/후반 프롬프트 품질 비교 | "재검진" 기능의 핵심 — 성장 서사 |

### 회사용 셀링 포인트 (보안 코칭)

| 항목 | 측정 방법 | 의미 |
|---|---|---|
| **민감정보 포함 검사** | 프롬프트 내 API 키·비밀번호 패턴 (정규식, 로컬에서만) | 점수가 아니라 경고로 표시. "검진이 보안 습관까지 잡아준다" — 제안서에서 강력함 |

### 채택 기준

- 축은 5~7개를 넘기지 말 것 (리포트가 성적표처럼 무거워짐). 신규 항목은 기존 A~E 축의 세부 근거로 흡수하는 게 우선.
- 예: 맥락 첨부율·산출물 명시율 → A축 세부지표, 중단 빈도·첫 시도 폐기율 → B축 세부지표, 병렬 활용·환경 구축 → E축 세부지표.
- 독립 축으로 승격할 가치가 있는 것: **보안 습관** (점수 아닌 별도 소견란), **성장 추적** (재검진 시에만).

## 2. 성능 / 제한 설계

### 병목 지점

1. **파싱 시간**: analyze.py는 라인 단위 스트리밍이라 수백 MB도 돌긴 하지만, 헤비 유저(수천 세션·수 GB)는 수십 초 걸릴 수 있음.
2. **LLM 채점 컨텍스트**: 진짜 병목. 세션 수 × 샘플 프롬프트가 컨텍스트를 초과하면 채점 품질이 무너짐.

### 제한 정책 (권장 기본값)

| 항목 | 기본값 | 근거 |
|---|---|---|
| 분석 기간 | 최근 30일 | 건강검진 컨셉과 일치 ("최근 상태" 진단). `--all` 옵션으로 해제 |
| 세션 수 상한 | 최근 50개 | 초과 시 최신순 50개 + "N개 제외됨" 명시 |
| 세션당 샘플 | 첫 프롬프트 1 + 교정 3 (현행 유지) | |
| 전체 샘플 총량 | 프롬프트 40개 / 각 300자 (~12K자) | 초과 시 세션당 균등 샘플링으로 줄임 |
| 세션 파일 크기 | 10MB 초과 세션은 집계만, 샘플 제외 | 서브에이전트 로그 등 비정상 케이스 방어 |

### 증분 캐시 (헤비 유저 대응)

- 세션별 요약을 `~/.claude/ai-checkup-cache.json`에 저장.
- 재진단 시 mtime이 바뀐 세션만 재파싱 → 두 번째 진단부터는 몇 초 안에 완료.
- 부수 효과: 캐시에 과거 진단 점수를 남기면 **재검진 비교**("지난달 대비 E축 +1") 기능이 공짜로 생김.

## 3. 우선순위 제안

1. **제한 정책** (30일/50세션/샘플 상한) — 배포 전 필수. 헤비 유저 한 명이 "안 끝나요" 하면 신뢰 끝.
2. **민감정보 검사** — 구현 반나절, 제안서 임팩트 최대.
3. **A/B/E축 세부지표 흡수** (맥락 첨부율, 산출물 명시율, 중단 빈도) — 채점 근거가 풍부해져 점수 흔들림도 감소.
4. **증분 캐시 + 재검진 비교** — 채택 후 "계속 쓰는 이유"를 만드는 기능.
